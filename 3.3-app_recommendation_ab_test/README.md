<b>Название:</b> Разработка и анализ A/B‑теста для нового алгоритма рекомендаций в развлекательном приложении.

<b>Цель:</b> проверить гипотезу о том, что новый алгоритм рекомендаций повышает вовлечённость пользователей в развлекательном приложении с «бесконечной» лентой контента, и проанализировать результаты эксперимента.

Приложение использует две модели монетизации:
- платная подписка (просмотр ленты без рекламы);
- бесплатная версия с демонстрацией рекламы.

<b>Источники данных:</b> три CSV‑таблицы с данными о сессиях пользователей:
- `sessions_project_history.csv` — исторические данные (11.08.2025–23.09.2025);
- `sessions_project_test_part.csv` — данные за первый день A/B‑теста (14.10.2025);
- `sessions_project_test.csv` — полные данные за весь период A/B‑теста (14.10.2025–02.11.2025).

<b>Ключевые поля в таблицах:</b>
|Поле |Описание|
|-----|--------|
|user_id |Идентификатор пользователя|
|session_id |Идентификатор сессии|
|session_date |Дата сессии|
|session_start_ts |Дата и время начала сессии|
|install_date |Дата установки приложения|
|session_number |Порядковый номер сессии для пользователя|
|registration_flag |Статус регистрации пользователя|
|page_counter |Количество просмотренных страниц в сессии|
|region |Регион пользователя|
|device |Тип устройства|
|test_group |Принадлежность к тестовой группе|

<b>Задачи проекта:</b>
- рассчитать параметры A/B‑теста;
- оценить корректность проведения эксперимента (в т. ч. распределение пользователей по группам, репрезентативность выборки);
- проанализировать результаты теста — сравнить ключевые метрики вовлечённости (например, количество просмотренных страниц) между контрольной и тестовой группами;
- сформулировать выводы о влиянии нового алгоритма на поведение пользователей и дать рекомендации по его внедрению.

<b>Инструменты:</b> Jupyter Notebook (Python).
