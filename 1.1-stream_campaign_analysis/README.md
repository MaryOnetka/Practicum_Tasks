<b>Название:</b> Анализ статистики прослушиваний для маркетинговой кампании сервиса «Поток».

<b>Цель проекта:</b> выбрать наиболее перспективный город (Москва или Санкт‑Петербург) для проведения крупной маркетинговой кампании музыкального стримингового сервиса «Поток».

<b>Задача аналитика:</b> выгрузить и структурировать данные о прослушивании музыки в Москве и Санкт‑Петербурге из базы данных PostgreSQL для поддержки принятия бизнес‑решений.

<b>Исходные данные:</b>

- сырой, неструктурированный набор данных о прослушиваниях в двух городах;

- источник данных: база данных PostgreSQL сервиса «Поток».

<b>Ключевые задачи:</b>

- подключиться к СУБД PostgreSQL и выгрузить данные по прослушиваниям в Москве и Санкт‑Петербурге;

- сформировать набор SQL‑запросов для получения релевантной информации (по активности пользователей, популярности жанров, динамике прослушиваний и т. д.);

- подготовить структурированные данные, позволяющие сравнить пользовательскую активность в двух городах;

- обеспечить команду сервиса данными для оценки потенциальной эффективности маркетинговой кампании в каждом из городов.

База данных состоит из одной таблицы `songs`. Вот её описание:

|Поле  |Тип данных  |Описание  |
|-----|-----|----------|
|id  |integer  |Уникальный идентификатор записи  |
|user_id  |integer  |Идентификатор пользователя |
|track  |character  |Название трека  |
|artist  |character  |Исполнитель  |
|genre  |character  |Жанр музыки  |
|city  |character  |Город прослушивания: Москва или Санкт-Петербург |
|day  |character  |День прослушивания  |
|duration  |integer  |Длительность прослушивания в секундах  |
|listen  |numeric  |Процент прослушивания трека  |
