<b>Название:</b> Анализ данных сервиса проката самокатов GoFast и проверка бизнес‑гипотез.

<b>Цель:</b> проанализировать демографию пользователей и особенности использования самокатов, оценить выгоду от распространения платной подписки, проверить ключевые бизнес‑гипотезы для принятия решений по маркетинговым акциям.

<b>Источники данных:</b>
- `users_go.csv` — данные о пользователях (ID, имя, возраст, город, тип подписки);
- `rides_go.csv` — данные о поездках (ID пользователя, расстояние, длительность, дата);
- `subscriptions_go.csv` — условия подписок (тип, цена минуты, старт поездки, ежемесячный платёж).

<b>Этапы работы:</b>
- загрузка данных: чтение CSV‑файлов в датафреймы Pandas, предварительный просмотр и изучение общей информации;
- предобработка данных:
  * приведение столбца date к типу даты;
  * создание столбца с номером месяца;
  * проверка и обработка пропусков и дубликатов;
- исследовательский анализ данных (EDA): визуализация и описание:
  * распределения городов;
  * соотношения пользователей с подпиской/без подписки;
  * возраста пользователей;
  * расстояния и длительности поездок;
- объединение данных:
  * слияние таблиц о пользователях, поездках и подписках;
  * разделение на группы по наличию подписки;
  * визуализация расстояний и времени поездок для обеих групп;
- расчёт выручки:
  * группировка данных по пользователю, имени, типу подписки и месяцу;
  * расчёт суммарного расстояния, количества поездок, времени и выручки по месяцам для каждого пользователя;
- проверка гипотез:
  * сравнение длительности поездок у пользователей с подпиской и без неё;
  * анализ расстояния одной поездки (сравнение с эталонным значением 3130 м для подписчиков);
  * сравнение помесячной выручки от пользователей с подпиской и без подписки;
- анализ распределений и моделирование:
  * расчёт средней длительности поездки и стандартного отклонения;
  * моделирование длительности поездки с помощью нормального распределения;
  * оценка доли поездок длительностью более 30 минут (для предложения скидки);
  * определение критической дистанции (90‑й процентиль) для введения дополнительной платы за длительные поездки.

<b>Инструменты:</b> Jupyter Notebook (Python) из тренажёра (тетрадь с задачами и описанием из тренажёра, код написан самостоятельно).
