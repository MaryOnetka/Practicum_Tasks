<b>Цель проекта:</b> создать интерактивный дашборд на основе исторических данных конференций TED — чтобы помочь компании организовать запоминающееся первое мероприятие с учётом успешного опыта прошлых выступлений.

<b>Бизнес‑задача:</b> дать заказчику ответы на ключевые вопросы для планирования конференции:
- какие темы наиболее популярны;
- как зрители реагируют на выступления (аплодисменты, смех, просмотры);
- какое среднее количество выступающих на конференции;
- какова средняя длина выступления;
- кто из выступающих наиболее интересен аудитории.

<b>Источник данных:</b> датасет с информацией о выступлениях на конференциях TED (3 таблицы):

Таблица `events` содержит информацию о конференциях.
|Поле |Описание|
|-----|--------|
|conf_id |Идентификатор конференции/Первичный ключ таблицы|
|event_name |Название конференции|
|country |Страна проведения конференции|

Таблица `speakers` содержит информацию о выступающих.
|Поле |Описание|
|-----|--------|
|author_id |Идентификатор выступающего/Первичный ключ таблицы|
|speaker_name |Имя выступающего|
|speaker_occupation |Род деятельности выступающего|
|speaker_description |Краткое описание выступающего|

Таблица `talks` содержит информацию о выступлениях.
|Поле |Описание|
|-----|--------|
|talk_id |Идентификатор выступления/Первичный ключ таблицы|
|url |Интернет-ресурс, где выступление находится в открытом доступе|
|title |Название выступления|
|description |Краткое описание выступления|
|film_date |Дата видеозаписи выступления|
|duration |Длительность выступления|
|views_count |Количество просмотров выступления|
|main_tag |Основной тег выступления|
|speaker_id |Идентификатор выступающего/Внешний ключ, который связан со столбцом author_id таблицы speakers|
|laughter_count |Число случаев, когда в зале звучал смех|
|applause_count |Число случаев, когда в зале звучали аплодисменты|
|language_sp |Язык, на котором велось выступление|
|event_id |Идентификатор конференции/Внешний ключ, который связан со столбцом conf_id таблицы events|

<b>Ключевые метрики для дашборда:</b>
- общие показатели: количество конференций, выступлений, уникальных выступающих, тегов;
- по конференциям: средняя длительность выступления, макс. количество просмотров, общее число аплодисментов, количество выступающих;
- топ‑10 самых «смешных» выступлений (по laughter_count);
- топ‑20 самых популярных тегов (main_tag);
- род деятельности выступающих с наибольшим числом аплодисментов.

<b>Функциональные требования к дашборду:</b>
- детализация данных от страны → к конференциям → к выступлениям;
- интерактивные селекторы (фильтры) по:
  * стране;
  * названию конференции;
  * тегу выступления;
  * дате записи (film_date);
  * возможность детализировать информацию по каждому выступлению при обнаружении интересного инсайта.

<b>Этапы работы:</b>
- разведочный анализ данных (изучение структуры, качества, объёма);
- объединение таблиц с использованием JOIN;
- расчёт ключевых метрик и формирование витрин данных;
- проектирование и разработка дашборда с интерактивной визуализацией;
- настройка фильтров и механизмов детализации.

<b>Инструменты:</b> BI‑система (Yandex Datalens) для построения дашборда.
