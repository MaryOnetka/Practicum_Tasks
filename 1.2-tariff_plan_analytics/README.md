<b>Название:</b> Анализ активности клиентов и эффективности тарифных планов «Мегасеть».

<b>Цель проекта:</b> проанализировать использование услуг связи клиентами компании «Мегасеть» по тарифным планам Smart и Ultra, оценить долю клиентов с перерасходом лимитов и подготовить данные для обновления линейки тарифов и разработки спецпредложений.

<b>Бизнес‑задача:</b>
- обновить тарифные планы под актуальные запросы клиентов;
- разработать систему специальных предложений для самых активных пользователей.

<b>Объект анализа:</b> два тарифных плана компании «Мегасеть»:

`Smart`: ежемесячный платёж — 550 руб., включает 500 мин звонков, 15 ГБ интернета, 50 сообщений; стоимость сверх лимита: минута — 3 руб., ГБ — 200 руб., сообщение — 3 руб.

`Ultra`: ежемесячный платёж — 1950 руб., включает 3000 мин звонков, 30 ГБ интернета, 1000 сообщений; стоимость сверх лимита: минута — 1 руб., ГБ — 150 руб., сообщение — 1 руб.

<b>Задачи аналитика:</b>
- выгрузить данные об активности клиентов за календарный месяц: суммарная длительность звонков (с округлением вверх), объём интернет‑трафика, количество сообщений.
- пассчитать текущие траты клиентов с учётом тарифных лимитов и стоимости услуг сверх плана.
- для каждого тарифа рассчитать средние траты клиентов (учитывая только действующих пользователей).
- среди активных клиентов выделить тех, кто пользуется услугами сверх тарифного плана, и посчитать:
  * средние расходы для каждого тарифа;
  * среднее значение переплаты для каждого тарифа.

<b>Этапы работы:</b>
- проверка данных: поиск и устранение пропущенных значений и дубликатов.
- подготовка данных: формирование набора данных об активности клиентов (звонки, интернет‑трафик, сообщения).

<b>Анализ и расчёты:</b> написание SQL‑запросов для решения поставленных задач и расчёта ключевых метрик.

База данных состоит из нескольких таблиц схемы <b>`telecom`</b>: `users`, `calls`, `internet`, `messages` и `tariffs`.

Таблица `users` содержит информацию о пользователях.
|Поле	|Описание|
|-----|--------|
|user_id	|Идентификатор пользователя/Первичный ключ|
|age	|Возраст пользователя|
|churn_date	|Дата окончания договора/Если договор действующий, то поле содержит пустую строку|
|city	|Город проживания пользователя|
|first_name	|Имя клиента|
|last_name	|Фамилия клиента|
|reg_date |Дата регистрации пользователя|
|tariff	|Тариф пользователя/Внешний ключ, связан с первичным ключом `tariff_name` таблицы `tariffs`|

Таблица `calls` содержит информацию о звонках.
|Поле	|Описание|
|-----|--------|
|id	|Идентификатор звонка/Первичный ключ|
|call_date	|Дата звонка|
|duration	|Длительность звонка в минутах|
|user_id	|Идентификатор пользователя/Внешний ключ, связан с первичным ключом `user_id` таблицы `users`|

Таблица `internet` содержит информацию о длительности интернет-сессий.
|Поле	|Описание|
|-----|--------|
|id	|Идентификатор сессии/Первичный ключ|
|mb_used	|Количество мегабайтов интернет-трафика, потраченного за сессию|
|session_date	|Дата сессии|
|user_id	|Идентификатор пользователя/Внешний ключ, связан с первичным ключом `user_id` таблицы `users`|

Таблица `messages` содержит информацию об отправленных сообщениях.
|Поле	|Описание|
|-----|--------|
|id	|Идентификатор сообщения/Первичный ключ|
|message_date	|Дата сообщения|
|user_id	|Идентификатор пользователя/Внешний ключ, связан с первичным ключом `user_id` таблицы `users`|

Таблица `tariffs` содержит информацию о тарифных планах.
|Поле	|Описание|
|-----|--------|
|tariff_name	|Название тарифа/Первичный ключ|
|messages_included	|Число бесплатных сообщений, которые входят в тариф|
|mb_per_month_included	|Количество мегабайтов интернет-трафика, которое входит в тариф|
|minutes_included	|Число минут разговора, которое входит в тариф|
|rub_monthly_fee	|Месячная абонентская плата в рублях|
|rub_per_gb	|Стоимость дополнительного гигабайта интернет-трафика сверх тарифного пакета в рублях (1 гигабайт = 1024 мегабайта)|
|rub_per_message	|Стоимость отправки сообщения сверх тарифного пакета в рублях|
|rub_per_minute	|Стоимость минуты разговора сверх тарифного пакета в рублях|
