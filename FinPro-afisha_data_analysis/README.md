<b>Цель проекта:</b> провести комплексный анализ данных сервиса Яндекс Афиша, выявить закономерности в изменении пользовательского спроса, сезонности и активности пользователей, а также предоставить команде продукта инструменты и инсайты для принятия решений перед периодом распродаж и новогодних акций.

<b>Часть 1.</b> Анализ данных с помощью SQL и создание дашборда в Yandex DataLens.

<b>Задача:</b> разработать аналитический дашборд для отслеживания ключевых бизнес‑показателей, популярности мероприятий и структуры выручки в разрезе категорий событий и устройств.

<b>Источники данных:</b> база данных PostgreSQL `data-analyst-afisha`, схема `afisha`, таблицы:
|Название таблицы |Описание таблицы |
|-----------------|-----------------|
|purchases |Информация о заказах билетов|
|events |Данные о мероприятиях|
|venues |Сведения о площадках|
|regions |Список регионов|
|city |Список городов|

<b>Ключевые метрики для дашборда:</b>
- итоговая выручка за период;
- общее количество заказов;
- средняя выручка с одного заказа;
- число уникальных покупателей;
- недельная динамика выручки, количества заказов и средней выручки с заказа;
- структура выручки по типам устройств и мероприятий;
- топ‑сегменты по выручке: регионы, события, площадки (с расчётом выручки, количества заказов, среднего числа билетов в заказе, средней выручки с билета).

<b>Требования к дашборду:</b>
- визуализация индикаторов и линейных графиков;
- таблицы с топ‑сегментами;
- фильтры по валюте (по умолчанию — рубли), дате (01.06–31.10.2024), типу событий и региону.

<b>Часть 2.</b> Исследовательский анализ данных и проверка гипотез в Python.
<b>Задачи:</b>
- предобработка данных:
  * проверка на пропуски и дубликаты;
  * нормализация категориальных значений;
  * обработка выбросов (фильтрация по 99‑му процентилю для revenue);
  * конвертация валюты в рубли (revenue_rub);
  * расчёт выручки с одного билета (one_ticket_revenue_rub);
  * создание полей month и season (лето, осень, зима, весна);
- исследовательский анализ:
  * анализ сезонности: динамика заказов по месяцам, сравнение лета и осени по категориям (тип мероприятия, устройство, возрастной рейтинг);
  * изучение осенней активности пользователей: динамика по дням, DAU, среднее число заказов на пользователя, средняя стоимость билета, недельная цикличность (будни/выходные);
  * выявление популярных событий и партнёров: распределение мероприятий по регионам, активность билетных партнёров;
- проверка гипотез (на данных за осень 2024):
  * H₀: среднее количество заказов на пользователя одинаково для мобильных и стационарных устройств;
    
    H₁: среднее количество заказов выше для мобильных устройств;
  * H₀: среднее время между заказами одинаково для мобильных и стационарных устройств;
    
    H₁: среднее время между заказами больше для мобильных устройств;
- формулировка выводов и рекомендаций:
  * обобщение результатов анализа (востребованные мероприятия, изменения популярности осенью, динамика среднего чека, лидеры среди регионов и партнёров);
  * интерпретация результатов проверки гипотез;
  * практические рекомендации для команды продукта.

<b>Используемые инструменты:</b>
- DBeaver - подключение к БД и выполнение SQL‑запросов;
- Yandex DataLens - создание интерактивного дашборда с фильтрацией;
- Jupyter Notebook (Python - библиотеки pandas, numpy, matplotlib, seaborn, scipy) — предобработка, исследовательский анализ, визуализация, статистическая проверка гипотез, документирование и выполнение анализа.
